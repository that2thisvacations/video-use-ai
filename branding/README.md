# Branding

This folder holds TravelBuddy branding assets and notes.

The current demo workflow only consumes placeholder metadata here. No render logic uses these files yet.

## Asset Areas

- `assets/watermarks/` for watermark files and notes
- `assets/endcards/` for end-card artwork and notes
- `assets/fonts/` for brand font references and notes

The next implementation stage should map these folders into the demo wrapper, then into render hooks.
